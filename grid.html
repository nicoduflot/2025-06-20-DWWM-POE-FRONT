Les aires de grille css

Les aires de grilles permettent de définir des zone dans la pages, selon une grille.

Par exemple, en smartphone le site s'affiche ainsi

Quand la largeur de l'affichage de l'appareil atteint ou dépasse 900px, le site s'affiche ainsi.

Il faut déclarer comme ici, le body en display: grid.

Le display: grid permet de définir des modèles de colonnes

en smartphone, on ne déclare qu'une colonne, chacun des éléments enfants directs de body occupera une ligne dans une colonne, avec la propriété grid-template-columns.

Il faut pourtant définir les aires de la grille avec la propriété grid-template-areas.

En smartphone :

body{
  padding: var(--gutter-x);
  display: grid;
  grid-template-columns: 1fr;
  grid-template-areas:
  'entete'
  'nav'
  'content'
  'info'
  'footer'
  ;
  gap: var(--gutter-x);
}

Il faut ensuite associer chacun des éléments enfants direct à une aire du template

#header{
  grid-area: entete;
}
#main-nav{
  grid-area: nav;
}
#content{
  grid-area: content;
}
#info{
  grid-area: info;
}
#footer{
  grid-area: footer;
}

Pour l'affichage au dessus de 900px, il va falloir changer le nombre de colonne.

on peut indique une largeur fixe à la première colonne et une fraction du reste pour la deuxième : la deuxième aura sa taille qui changera selon la largeur de l'écran moins la largeur fixe de la première colonne

Il faudra aussi redéclarer les aires de grille : deux pour l'entete et la nav car ils prennent toute la largeur de l'écran, on déclare cette fois-ci info avant content et les deux aires sur la même ligne, le navigateur placera avant le contenu l'air info en lui imposant une taille fixe alors que content occupera tout l'espace restant. Enfin on déclare le footer lui aussi sur deux colonnes.

@media screen and (min-width: 900px) {
  body{
    grid-template-columns: 250px 1fr;
    grid-template-areas:
    'entete entete'
    'nav nav'
    'info content'
    'footer footer'
    ;
  }
}
